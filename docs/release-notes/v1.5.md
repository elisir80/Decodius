# Decodius v1.5

Questa release aggiorna Decodius rispetto alla v1.4 con un primo avvio piu' guidato, packaging multipiattaforma e correzioni specifiche per il bundle macOS.

## Novita'

- Aggiunto controllo automatico di Ollama al primo avvio: Decodius verifica se il servizio locale risponde su `127.0.0.1:11434`.
- Se Ollama manca o non risponde, l'app mostra un dialog dedicato con azioni diverse per piattaforma:
  - Windows: configurazione automatica con `setup_cervello.ps1`;
  - macOS: apertura download ufficiale e avvio di `Ollama.app` quando presente;
  - Linux: istruzioni di installazione ufficiali e avvio di `ollama serve` quando disponibile.
- Aggiunto pulsante di ricontrollo dello stato Ollama dopo installazione o avvio del servizio.
- Aggiunta gestione del modello Ollama dall'interfaccia impostazioni, con persistenza in configurazione utente.
- Migliorato il supporto alle risorse runtime nel bundle macOS: modello, prompt, knowledge base e script vengono cercati anche in `Contents/Resources`.
- Corretto il packaging macOS: il bundle ora viene finalizzato con `macdeployqt`, evitando il caricamento simultaneo di framework Qt di sistema/Homebrew e framework interni all'app.
- Aggiunti workflow GitHub Actions dedicati a Decodius per produrre gli artefatti di release separatamente da Decodium.

## Artefatti previsti

- Installer Windows x64.
- DMG macOS.
- AppImage Linux x86_64.
- AppImage Linux aarch64 quando e' disponibile il runner GitHub/Decodius per architettura ARM64.

## Note

Ollama resta un prerequisito funzionale: Decodius puo' guidare l'utente nell'installazione o nell'avvio, ma il modello deve essere disponibile tramite Ollama per generare risposte.
